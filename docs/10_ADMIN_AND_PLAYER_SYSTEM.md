## مقدمة

يتكون مشروع DOSSIER X من تطبيقين منفصلين تمامًا، لكنهما يستخدمان نفس محرك القضايا (Case Engine).
هما ليسا نفس البرنامج، ولا يشارك اللاعب أي واجهة أو صلاحية مع الأدمن.

```
                 DOSSIER X
        ┌────────────────────────┐
        │   CASEFORGE STUDIO      │
        │      (Admin Only)       │
        └───────────┬─────────────┘
                    │
         إنشاء وتصدير القضايا
                    │
          DX001.casepack
          DX002.casepack
          DX003.casepack
                    │
                    ▼
        ┌────────────────────────┐
        │     DOSSIER X PLAYER    │
        │      (Customer)         │
        └────────────────────────┘
```

## أولاً: PLAYER APPLICATION

هذا هو البرنامج الذي يشتريه أو يستخدمه العميل.
اللاعب لا يرى أي أدوات إنشاء أو تعديل. كل ما يراه هو تجربة التحقيق.

### أول تشغيل

عند فتح المنصة لأول مرة تظهر شاشة افتتاحية.

```
+------------------------------------------------+
|                DOSSIER X                        |
|      Every Detail Tells a Story                |
|                                                |
| Detective Name                                 |
| [__________________________]                   |
|                                                |
| Language                                       |
| (•) العربية   ( ) English                      |
|                                                |
|        [ START INVESTIGATION ]                 |
+------------------------------------------------+
```

بعد الضغط على Start Investigation:
- يُحفظ الاسم.
- تُحفظ اللغة.
- يُنشأ ملف محقق محلي.
- ينتقل إلى لوحة التحكم.

### لوحة التحكم (Dashboard)

تتكون من: Sidebar ثابت، Detective Profile، Archive، آخر قضية تم لعبها،
الإنجازات الأخيرة، الشهادات.

### Detective Profile

يعرض: صورة رمزية، اسم المحقق، الرتبة، عدد القضايا المنجزة، نقاط الخبرة،
نسبة التقدم.

مثال:
```
Detective Ahmed
Rank: Senior Investigator
XP: 2450
Solved Cases: 8
```

### Archive

الأرشيف يعرض جميع القضايا. كل بطاقة تحتوي على: الغلاف، اسم القضية، مستوى
الصعوبة، الوقت المتوقع، عدد الأدلة، عدد الوثائق، حالة القضية.

الحالات الممكنة: Locked / Owned / In Progress / Completed

إذا كانت القضية مقفلة، يظهر:
```
CLASSIFIED
ACCESS DENIED
This investigation requires clearance.
```
وعند الضغط: يفتح مربع شراء.

إذا كانت القضية مملوكة، يظهر زر: Continue Investigation

إذا كانت منتهية، يظهر: Completed / View Certificate

### داخل القضية

القائمة الجانبية لا تتغير. لكن المحتوى الأوسط يتغير.

الأقسام: Case File, Crime Scene, Evidence Locker, Suspects, Witnesses,
Timeline, Notebook, Questions, Final Report

### ماذا يستطيع اللاعب؟

يمكنه: قراءة الملفات، تكبير الصور، تشغيل الصوت، كتابة الملاحظات، وضع
Bookmark، الإجابة على الأسئلة، تحميل الشهادة، استيراد قضية، تصدير تقدمه.

### ماذا لا يستطيع؟

لا يستطيع: تعديل الأدلة، حذف الشهود، إضافة مشتبه، تغيير الحل، رؤية
الإجابات، تعديل ملفات القضية، تعديل الشهادة، تغيير الصور.

كل المحتوى Read Only.

### الحفظ

كل حركة تحفظ تلقائياً. مثلاً: يفتح دليل → يحفظ. يكتب ملاحظة → يحفظ. يقرأ
شاهد → يحفظ.

### الشهادة

بعد النجاح، ينشئ النظام: PDF, PNG, JSON — ويحفظها داخل Certificates.

### الإنجازات

كل إنجاز يفتح تلقائياً. مثال: First Case, Perfect Score, Master Detective

### إعدادات اللاعب

اللغة. استيراد التقدم. تصدير التقدم. إعادة تعيين. حول البرنامج.

## ثانياً: CASEFORGE STUDIO

هذا تطبيق منفصل تماماً. لا يراه اللاعب.

### الهدف

إنشاء القضايا الجديدة. لا يحتاج أي معرفة برمجية. كل شيء يتم عبر واجهات
رسومية.

### Dashboard

يعرض: عدد القضايا، آخر تعديل، عدد الإصدارات، آخر تصدير.

### Cases

قائمة بجميع القضايا: DX001, DX002, DX003, DX004, + Create New Case

### Create Case

ينتقل إلى معالج إنشاء القضية. المراحل:
1. General Information
2. Victim
3. Crime Scene
4. Evidence
5. Suspects
6. Witnesses
7. Timeline
8. Documents
9. Questions
10. Solution
11. Preview
12. Export

### General Information

يدخل الأدمن: اسم القضية بالعربية، اسم القضية بالإنجليزية، الوصف، الغلاف،
الصعوبة، الوقت المتوقع، نوع الجريمة، الكلمات المفتاحية، رقم الإصدار.

### Victim

يرفع: الصورة، الاسم، العمر، الوظيفة، سبب الوفاة، وقت الوفاة، مكان الوفاة.

### Crime Scene

يرفع: الصور، خريطة المكان، وصف المكان، الطقس، الإضاءة، وقت وصول الشرطة.

### Evidence

زر: Add Evidence

لكل دليل: رقم الدليل، الصورة، الفئة (بصمة، DNA، سلاح…)، الوصف، الموقع،
مرتبط بأي شخص، مرتبط بأي حدث.

### Suspects

زر: Add Suspect

لكل مشتبه: الصورة، الاسم، العمر، الوظيفة، العلاقة بالضحية، الدافع،
الخلفية، ملاحظات.

### Witnesses

زر: Add Witness

لكل شاهد: الصورة، الاسم، الإفادة، مستوى الموثوقية، وقت المقابلة، الأدلة
المرتبطة.

### Timeline

يبني خطاً زمنياً. كل حدث يحتوي على: الوقت، الوصف، الأشخاص، الأدلة، الموقع.

### Documents

الأدمن يستطيع إنشاء: تقرير الشرطة، تقرير مسرح الجريمة، تقرير الطب الشرعي،
تقرير البصمات، تقرير DNA، تقرير الألياف، تقرير السموم، الرسائل، الملاحظات.

### Questions

يبني الاختبار النهائي. أنواع الأسئلة: اختيار من متعدد، اختيار واحد،
ترتيب زمني، اختيار دليل، كتابة نص.

### Solution

يحدد: القاتل، الدافع، أداة الجريمة، أهم دليل، الإجابات الصحيحة، الدرجة
الكاملة.

هذه الصفحة لا يراها اللاعب أبدًا.

### Preview

زر: Preview Investigation — يفتح نسخة اللاعب، تماماً كما سيراه العميل.
إذا وجد الأدمن خطأً يعود ويعدله.

### Export

يضغط: Generate Case

فينتج: DX005.casepack

الحزمة تحتوي على: جميع ملفات JSON، جميع الصور، جميع الوثائق، جميع
الترجمات، ملفات الصوت، الغلاف، الترخيص.

ثم يرفعها إلى Gumroad.

### دورة حياة القضية

```
Create → Edit → Preview → Export → Upload to Gumroad →
Customer Purchase → Import into Player → Play → Certificate → Completed
```

### قواعد الفصل بين التطبيقين

- لا يمكن للاعب فتح CASEFORGE Studio.
- لا يمكن للأدمن تعديل تقدم اللاعبين.
- لا توجد قاعدة بيانات مشتركة.
- التواصل الوحيد بين التطبيقين هو ملف .casepack.
- أي تحديث للقضية يصدر كإصدار جديد (Versioned Package).
- لا تُعدَّل ملفات القضايا بعد تثبيتها؛ تُنشأ نسخة جديدة عند كل تحديث.

## توصية معمارية أخيرة

```
dossier-x/
├── apps/
│   ├── player/          # منصة اللاعب
│   └── caseforge/       # برنامج إنشاء القضايا
├── packages/
│   ├── ui/              # المكونات المشتركة
│   ├── engine/          # محرك القضايا
│   ├── storage/         # IndexedDB
│   ├── certificates/
│   ├── achievements/
│   └── i18n/
├── docs/                # جميع الوثائق
├── design/              # الصور المرجعية والتصميم
└── templates/           # قوالب القضايا والوثائق
```

بهذا الهيكل سيستخدم كل من Player وCASEFORGE Studio نفس محرك القضايا ونفس
نظام التصميم، بينما يبقى كل تطبيق مستقلًا في واجهته ووظيفته.
